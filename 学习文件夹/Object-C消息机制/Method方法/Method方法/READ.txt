###########SEL与IMP##############
SEL----可以把它理解为“方法名的某种映射结果”
Objective-C在编译的时候，会根据方法的名字(包括参数序列)，生成一个用来区分这个方法的唯一的一个ID，这个ID就是SEL类型的。需要注意的是，只要方法的名字(包括参数序列)相同，那么它们的ID都是相同的。就是说，不管是超类还是子类，不管是有没有超类和子类的关系，只要名字相同那么ID就是一样的。

从效率的角度上来说,执行的时候不是通过方法名字而是方法 ID 也就是一个整数来查找方法, 由于整数的查找和匹配比字符串要快得多,所以这样可以在某种程度上提高执行的效率。

在 程序执行的时候,我们可以方便的通过方法的名字,获取到方法的 ID 也就是我们所说的 SEL, 反之亦然。具体的使用方法如下:
[pre]1    SEL 变量名 = @selector(方法名字);2    SEL 变量名 = NSSelectorFromString(方法名字的字符串);3    NSString *变量名 = NSStringFromSelector(SEL 参数);[/pre]

IMP----才是函数指针,说白了IMP就是一个实现方法，给一个方法起个名字，实现动态调用。IMP是“implementation”的缩写，它是objective-C方法(method)实现代码块的地址，可像C函数一样直接调用。通常情况下我们是通过[object method:parameter]或objc_msgSend()方法向对象发送消息,然后Objective-C运行时(runtime)寻找匹配消息的IMP,然后调用它，但有些时候我们希望获取到IMP进行直接调用的。
